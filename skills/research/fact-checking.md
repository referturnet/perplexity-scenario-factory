# Fact Checking (Проверка фактов)

## Описание

Навык верификации информации и проверки достоверности фактов из веб-источников. Позволяет AI-агенту перекрёстно проверять утверждения через несколько независимых источников, оценивать надёжность источников, выявлять дезинформацию и формировать обоснованные выводы о достоверности данных.

## Связанные навыки

- `deep-research.md` — Глубокое исследование как основа для сбора источников
- `competitive-analysis.md` — Верификация данных о конкурентах
- `data-extraction.md` — Извлечение данных для сравнения
- `navigation.md` — Навигация по источникам для проверки

---

## Инструменты

### Доступные функции:

- **`search_web(queries)`** — Поиск по нескольким запросам для перекрёстной проверки
- **`navigate(url, tab_id)`** — Открытие первоисточников и официальных страниц
- **`get_page_text(query, tab_id)`** — Извлечение релевантного текста для анализа
- **`read_page(tab_id)`** — Анализ структуры страницы для оценки надёжности
- **`tabs_create(url)`** — Параллельная проверка в нескольких вкладках
- **`find(query, tab_id)`** — Поиск дат публикации, авторов, источников

## Алгоритм выполнения

### 1. Декомпозиция утверждения
- Разбить утверждение на отдельные проверяемые факты
- Выделить ключевые параметры: числа, даты, имена, события
- Определить тип утверждения: статистика, цитата, событие, научный факт
- Сформулировать 3-5 разных поисковых запросов для каждого факта

### 2. Сбор источников
- Искать через `search_web` с разными формулировками запросов
- Собрать минимум 3-5 независимых источников на каждый ключевой факт
- Приоритизировать: официальные сайты > академические > СМИ > блоги
- Зафиксировать URL, дату публикации, автора для каждого источника

### 3. Оценка качества источников
- Проверить домен: .gov, .edu, .org vs анонимные сайты
- Найти дату последнего обновления материала
- Установить авторство и его релевантную экспертизу
- Проверить наличие ссылок на первоисточники в самом материале
- Выявить возможный конфликт интересов

### 4. Перекрёстная верификация
- Сравнить данные из независимых источников между собой
- Выявить противоречия и расхождения
- Для статистики: найти оригинальное исследование/отчёт
- Для цитат: найти оригинальный контекст высказывания
- Для событий: проверить временную последовательность

### 5. Анализ контекста
- Убедиться, что факт не вырван из контекста
- Проверить актуальность данных (не устарели ли)
- Выявить возможные манипуляции с числами (выборка, методология)
- Найти опровержения или уточнения от первоисточника

### 6. Формирование вердикта
- Оценить совокупность доказательств
- Присвоить уровень достоверности: Подтверждено / Частично верно / Спорно / Опровергнуто / Недостаточно данных
- Указать все источники с оценкой их надёжности
- Описать ограничения проведённой проверки

## Примеры использования

### Пример 1: Проверка статистического утверждения
```
Утверждение: "Python используют 80% data scientists"

Действия:
1. search_web(["Python data scientists usage statistics 2024", 
   "programming languages data science survey", 
   "Python popularity machine learning"])
2. Найти первоисточник: Stack Overflow Survey, JetBrains Survey, Kaggle Survey
3. Открыть оригинальные отчёты, извлечь точные цифры
4. Сравнить: Kaggle 2023 = 74%, Stack Overflow 2023 = 68%

Вердикт: Частично верно — цифра завышена, реальные данные 68-74%
Источники: [ссылки на официальные отчёты]
```

### Пример 2: Проверка цитаты известной личности
```
Утверждение: "Илон Маск сказал 'AI опаснее ядерного оружия'"

Действия:
1. search_web(["Elon Musk AI nuclear weapons quote", 
   "Musk artificial intelligence dangerous quote source"])
2. Найти первоисточник: интервью, твит, выступление
3. Открыть оригинальный контекст
4. Проверить точность цитаты и контекст высказывания

Вердикт: Спорно — похожее высказывание было, но в другой формулировке
```

### Пример 3: Проверка новостного события
```
Утверждение: "Компания X объявила о банкротстве в марте 2024"

Действия:
1. search_web(["Company X bankruptcy 2024", "Company X financial news March 2024"])
2. Найти официальные пресс-релизы и юридические документы
3. Проверить несколько авторитетных новостных источников
4. Найти официальный сайт компании

Вердикт: Подтверждено / Опровергнуто — на основе совокупности источников
```

## Edge cases и обработка ошибок

### Проблема: Противоречивые данные в источниках
```markdown
Решение:
- Искать первоисточник (оригинальное исследование, официальный документ)
- Проверить даты публикаций — возможно данные обновились
- Отдать приоритет более свежим данным из авторитетных источников
- Указать противоречие в вердикте явно
```

### Проблема: Только один источник подтверждает факт
```markdown
Решение:
- Не делать окончательный вывод при единственном источнике
- Искать дополнительные независимые подтверждения
- Оценить надёжность единственного источника
- Вердикт: "Недостаточно данных" если источник один
```

### Проблема: Источники на разных языках дают разные данные
```markdown
Решение:
- Проверить, не являются ли расхождения результатом перевода
- Искать оригинальный язык первоисточника
- Учесть региональные различия в данных
- Указать языковой контекст в результатах
```

### Проблема: Сайт требует регистрации для доступа к данным
```markdown
Решение:
- Искать цитаты из закрытого материала в открытых источниках
- Проверять Google Scholar, ResearchGate для академических работ
- Использовать archive.org для архивных версий
- Указать ограничение доступа в описании источника
```

### Проблема: Факт касается очень свежих событий
```markdown
Решение:
- Учесть, что данные могут быть неполными
- Проверить официальные аккаунты в соцсетях организаций
- Искать breaking news в авторитетных СМИ
- Указать временное ограничение верификации
```

## Метрики качества

- **Количество источников**: минимум 3 независимых на ключевой факт
- **Качество источников**: доля первичных источников (>50%)
- **Актуальность**: давность данных (предпочтительно <2 лет)
- **Покрытие**: процент проверенных под-фактов от общего числа
- **Точность вердикта**: соответствие вывода совокупности доказательств
- **Прозрачность**: все источники задокументированы с датами

---

## Лучшие практики

1. **Разделять факты и интерпретации** — проверять только конкретные утверждения, не мнения
2. **Искать первоисточник всегда** — не останавливаться на пересказе
3. **Проверять дату публикации** — устаревшие данные часто вводят в заблуждение
4. **Использовать разные поисковые запросы** — одна формулировка может давать предвзятую выборку
5. **Документировать все источники** — даже те, что противоречат друг другу
6. **Признавать неопределённость** — "недостаточно данных" лучше ложного вердикта
7. **Проверять контекст цитат** — вырванная фраза меняет смысл
8. **Оценивать мотивацию источника** — кто стоит за публикацией и зачем
9. **Параллельная проверка в нескольких вкладках** — экономит время на независимую верификацию
10. **Фиксировать дату проверки** — факты могут измениться после верификации
