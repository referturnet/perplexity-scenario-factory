# Deep Research (Глубокое исследование)

## Описание
Навык проведения глубокого многоуровневого исследования темы с использованием множественных источников, перекрёстной проверки фактов и синтеза информации.

## Связанные навыки
- `navigation.md` — Переход к формам
- `data-extraction.md` — Проверка отправленных данных
- `retry-logic.md` — Retry при ошибках заполнения
- `fallback-strategies.md` — Альтернативные методы ввода

---

## Инструменты

### Доступные функции:

```markdown
- search_web(queries)
  Поиск информации в интернете
  
- navigate(url, tab_id)
  Переход на веб-страницу
  
- read_page(tab_id, filter="interactive")
  Получить элементы формы
  
- get_page_text(tab_id, query)
  Извлечение релевантного текста
  
- tabs_create(url)
  Создание новых вкладок для параллельного исследования
```

---

## Алгоритм выполнения

### 1. Планирование исследования
```markdown
1. Определить ключевые вопросы для исследования
2. Разбить тему на подтемы/аспекты
3. Составить список поисковых запросов
4. Определить критерии качества источников
```

### 2. Сбор информации
```markdown
1. Использовать search_web для каждого аспекта
2. Открыть релевантные источники в отдельных вкладках
3. Извлечь ключевую информацию с помощью get_page_text
4. Сохранить источники с цитированием
```

### 3. Перекрёстная проверка
```markdown
1. Сравнить информацию из разных источников
2. Проверить факты через авторитетные источники
3. Отметить противоречия
4. Искать первоисточники для важных утверждений
```

### 4. Углубление
```markdown
1. Следовать по ссылкам на связанные темы
2. Искать экспертные мнения и исследования
3. Проверять даты публикаций для актуальности
4. Искать статистику и данные
```

### 5. Синтез и структурирование
```markdown
1. Организовать информацию по категориям
2. Выявить ключевые выводы
3. Связать информацию из разных источников
4. Подготовить итоговый отчёт с цитированием
```

## Примеры использования

### Пример 1: Исследование технологии
```markdown
Запрос: "Проведи глубокое исследование технологии квантовых компьютеров"

Действия:
1. search_web(["quantum computing overview", "quantum computing applications", "quantum computing challenges"])
2. Открыть топ-5 источников в отдельных вкладках
3. get_page_text для каждой страницы с query="quantum computing capabilities"
4. search_web(["quantum computing vs classical computing"])
5. Найти академические источники и статистику
6. Составить структурированный отчёт
```

### Пример 2: Исследование рынка
```markdown
Запрос: "Исследуй рынок электромобилей в 2026 году"

Действия:
1. search_web(["EV market 2026", "electric vehicle sales statistics", "EV market leaders"])
2. tabs_create для параллельного изучения разных аспектов
3. get_page_text с query="market size" для страниц со статистикой
4. search_web(["EV charging infrastructure", "EV battery technology"])
5. Перекрёстная проверка цифр из разных источников
6. Синтез данных в комплексный отчёт
```

### Пример 3: Проверка фактов
```markdown
Запрос: "Проверь утверждение о том, что..."

Действия:
1. search_web с точной формулировкой утверждения
2. Искать первоисточник утверждения
3. search_web с альтернативными формулировками
4. Проверить через fact-checking сайты
5. Искать экспертные опровержения/подтверждения
6. Сформулировать вывод с источниками
```

## Edge cases и обработка ошибок

### Проблема: Противоречивая информация
```markdown
Решение:
- Отметить противоречие явно
- Указать источники обеих версий
- Оценить авторитетность источников
- Искать третьи независимые источники
- Проверить даты публикаций
```

### Проблема: Недостаточно источников
```markdown
Решение:
- Расширить поисковые запросы
- Использовать синонимы и альтернативные формулировки
- Искать на других языках (если применимо)
- Обращаться к академическим базам
- Отметить ограниченность доступной информации
```

### Проблема: Устаревшая информация
```markdown
Решение:
- Фильтровать результаты по дате
- Добавить "2026", "latest", "recent" к запросам
- Проверять даты на страницах
- Отмечать, когда информация устарела
- Искать обновления и изменения
```

### Проблема: Предвзятые источники
```markdown
Решение:
- Идентифицировать потенциальную предвзятость
- Искать альтернативные точки зрения
- Балансировать источники разных направлений
- Фокусироваться на фактах, а не мнениях
- Отмечать возможную предвзятость в отчёте
```

## Метрики качества

- **Количество источников**: минимум 5-7 различных источников
- **Разнообразие**: источники разных типов (новости, академические, отраслевые)
- **Актуальность**: приоритет свежим источникам
- **Авторитетность**: предпочтение проверенным источникам
- **Перекрёстная проверка**: ключевые факты подтверждены 2+ источниками

## Лучшие практики

1. **Начинать с общего обзора**, затем углубляться
2. **Использовать tabs_create** для параллельного исследования
3. **Цитировать все источники** с URL и датами
4. **Отмечать уровень уверенности** в информации
5. **Различать факты и мнения**
6. **Проверять автора и дату** публикации
7. **Следовать по цепочке** первоисточников для важных утверждений
8. **Организовать информацию** логически и структурированно
