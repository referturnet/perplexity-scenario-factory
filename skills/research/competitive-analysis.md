# Competitive Analysis (Конкурентный анализ)

## Описание
Навык проведения систематического анализа конкурентов, их продуктов, стратегий и позиций на рынке для выявления конкурентных преимуществ и возможностей.

## Связанные навыки
- `deep-research.md` — Глубокое исследование
- `data-extraction.md` — Извлечение данных
- `product-search.md` — Поиск продуктов
- `price-comparison.md` — Сравнение цен

---

## Инструменты

### Доступные функции:

```markdown
- search_web(queries)
  Поиск информации о конкурентах
  
- navigate(url, tab_id)
  Переход на сайты конкурентов
  
- get_page_text(tab_id, query)
  Извлечение ключевой информации
  
- tabs_create(url)
  Параллельное изучение конкурентов
```

---

## Алгоритм выполнения

### 1. Идентификация конкурентов
```markdown
1. Определить прямых конкурентов
2. Выявить непрямых конкурентов
3. Найти новых игроков на рынке
4. Составить список для анализа
```

### 2. Сбор информации
```markdown
1. Продукты и услуги
2. Ценообразование
3. Целевая аудитория
4. Маркетинговые стратегии
5. Сильные и слабые стороны
6. Отзывы клиентов
```

### 3. Анализ по категориям
```markdown
- Продуктовый портфель
- Ценовая политика
- Каналы продаж
- Маркетинг и PR
- Технологии и инновации
- Обслуживание клиентов
```

### 4. Сравнительная таблица
```markdown
Создать матрицу сравнения:
- Параметры продуктов
- Цены и условия
- Функциональность
- УТП (уникальные торговые предложения)
- Присутствие на рынке
```

## Примеры использования

### Пример 1: Анализ SaaS конкурентов
```markdown
Запрос: "Проанализируй главных конкурентов Notion"

Действия:
1. search_web(["Notion competitors", "best productivity apps 2026"])
2. Открыть сайты Notion, Coda, ClickUp, Monday.com
3. get_page_text для каждого с query="pricing features"
4. Составить сравнительную таблицу
5. Анализировать отзывы пользователей
6. Выявить gaps в функциональности
```

### Пример 2: Анализ e-commerce
```markdown
Запрос: "Сравни топ-3 интернет-магазина электроники"

Действия:
1. Идентифицировать лидеров рынка
2. tabs_create для параллельного анализа
3. Сравнить ассортимент
4. Проверить цены на аналогичные товары
5. Оценить условия доставки
6. Проанализировать программы лояльности
```

## Edge cases

### Проблема: Закрытая информация
```markdown
Решение:
- Использовать публичные источники
- Анализировать пресс-релизы
- Изучать отзывы и кейсы
- Проверять вакансии компаний
```

### Проблема: Много конкурентов
```markdown
Решение:
- Фокус на топ-5 игроков
- Группировка по сегментам
- Приоритизация по влиянию
```

## Метрики качества

- **Полнота**: все ключевые конкуренты учтены
- **Актуальность**: свежие данные
- **Объективность**: факты, не предположения
- **Структурированность**: удобный формат сравнения

---

## Лучшие практики

1. Регулярно обновлять данные
2. Использовать множественные источники
3. Проверять факты
4. Фокусироваться на измеримых параметрах
5. Выявлять тренды и паттерны
