# Skill: Data Extraction

## Описание

Навык извлечения структурированных данных из веб-страниц с помощью инструментов браузерной автоматизации.

## Применение

Извлечение информации с сайтов для:
- Сбора данных о продуктах
- Мониторинга цен
- Исследования рынка
- Автоматизации заполнения отчетов
- Проверки контента

---

## Основные техники

### 1. Использование read_page

```markdown
Читает DOM структуру страницы и возвращает accessibility tree

## Когда использовать:
- Нужен структурированный доступ к элементам
- Требуются ссылки на элементы (ref_id)
- Работа с интерактивными элементами

## Пример:
read_page(tab_id=1, filter="interactive")
Результат: список кнопок, ссылок, полей ввода с ref_id
```

### 2. Использование get_page_text

```markdown
Извлекает текстовое содержимое страницы

## Когда использовать:
- Нужен полный текст страницы
- Поиск конкретной информации по запросу
- Извлечение контента статей

## Пример с фильтром:
get_page_text(tab_id=1, query="цена продукта")
Вернет только текст, относящийся к ценам

## Пример без фильтра:
get_page_text(tab_id=1)
Вернет весь текст страницы (до 40K символов)
```

### 3. Использование find

```markdown
Поиск элементов по естественному описанию

## Когда использовать:
- Элемент не виден в последнем screenshot
- Нужно найти элемент по описанию
- Поиск по содержимому текста

## Пример:
find(tab_id=1, query="кнопка добавить в корзину")
Вернет до 20 подходящих элементов с координатами
```

## Стратегии извлечения данных

### Стратегия 1: Structured Extraction

**Цель**: Получить конкретные поля данных

```markdown
1. Открыть целевую страницу
2. Использовать read_page для получения структуры
3. Найти нужные элементы по ref_id или типу
4. Извлечь текст из каждого элемента
5. Структурировать в объект/таблицу

Пример - карточка товара:
- Название: text из h1
- Цена: text из элемента с классом price
- Описание: text из div с описанием
- Изображение: src атрибут img
```

### Стратегия 2: Query-Based Extraction

**Цель**: Найти информацию по ключевым словам

```markdown
1. Открыть страницу
2. Использовать get_page_text с query
3. Проанализировать возвращенный текст
4. Извлечь нужные данные

Пример - поиск спецификаций:
query="технические характеристики"
Извлечь: параметры, значения, единицы измерения
```

### Стратегия 3: Iterative Extraction

**Цель**: Собрать данные со множества элементов

```markdown
1. Получить список элементов (товары, статьи)
2. Для каждого элемента:
   a. Извлечь базовую информацию
   b. Опционально: перейти на детальную страницу
   c. Извлечь дополнительные данные
   d. Вернуться к списку
3. Агрегировать все данные

Пример - каталог товаров:
Для каждого товара в списке:
- ID, название, цена (из списка)
- Детали, отзывы (с детальной страницы)
```

## Работа с разными типами данных

### Таблицы

```markdown
## Техника:
1. Найти table элемент через read_page
2. Извлечь заголовки (th элементы)
3. Извлечь строки (tr элементы)
4. Для каждой строки извлечь ячейки (td)
5. Сформировать структуру: [{col1: val1, col2: val2}, ...]

## Альтернатива:
Использовать get_page_text если таблица простая
Парсить текст по шаблону
```

### Списки

```markdown
## Техника:
1. Найти контейнер списка (ul, ol, div)
2. Извлечь элементы списка (li, div)
3. Для каждого элемента извлечь текст/ссылки
4. Сформировать массив

## Маркеры для поиска:
- Bullet points (•, -, *)
- Нумерация (1., 2., 3.)
- Повторяющиеся паттерны в структуре
```

### Метаданные

```markdown
## Что извлекать:
- Заголовок страницы (title)
- Описание (meta description)
- Ключевые слова
- Open Graph данные
- Структурированные данные (JSON-LD)

## Техника:
Использовать read_page для meta тегов
Или get_page_text для видимых метаданных
```

## Best Practices

### 1. Оптимизация запросов

```markdown
✅ Правильно:
- Использовать get_page_text с query для длинных страниц
- Кешировать результаты read_page
- Минимизировать количество вызовов инструментов

❌ Неправильно:
- Читать всю страницу для одного значения
- Множественные read_page для одной страницы
- Игнорировать фильтры
```

### 2. Обработка ошибок

```markdown
- Проверять наличие элементов перед извлечением
- Обрабатывать случаи отсутствия данных
- Использовать значения по умолчанию
- Логировать проблемы для отладки
```

### 3. Валидация данных

```markdown
- Проверять формат извлеченных данных
- Конвертировать типы (строки в числа, даты)
- Удалять лишние пробелы и символы
- Проверять на полноту данных
```

### 4. Производительность

```markdown
- Извлекать данные пакетно где возможно
- Избегать избыточных переходов между страницами
- Использовать параллельные вкладки для независимых задач
- Оптимизировать глубину read_page
```

## Примеры сценариев

### Сценарий 1: Извлечение цены товара

```markdown
1. Navigate к странице товара
2. Wait для загрузки
3. get_page_text(query="цена")
4. Извлечь числовое значение из текста
5. Вернуть структуру: {price: 1299, currency: "RUB"}
```

### Сценарий 2: Сбор списка товаров

```markdown
1. Navigate к странице категории
2. read_page(filter="interactive")
3. Найти все карточки товаров
4. Для каждой карточки:
   - Извлечь название (text)
   - Извлечь цену (text)
   - Извлечь ссылку (href)
5. Вернуть массив: [{name, price, url}, ...]
```

### Сценарий 3: Извлечение контента статьи

```markdown
1. Navigate к статье
2. get_page_text()
3. Идентифицировать основной контент
4. Извлечь:
   - Заголовок
   - Автор
   - Дата
   - Основной текст
   - Изображения
5. Структурировать в объект
```

## Граничные случаи

### Динамический контент

```markdown
## Проблема:
Данные загружаются асинхронно через JavaScript

## Решение:
1. Wait после навигации
2. Проверить наличие нужных элементов
3. Дождаться полной загрузки
4. Retry если данные не появились
```

### Пагинация

```markdown
## Проблема:
Данные разбиты на страницы

## Решение:
1. Извлечь данные с текущей страницы
2. Найти кнопку "Следующая"/"Next"
3. Click и wait
4. Повторить пока есть страницы
5. Агрегировать все данные
```

### Бесконечная прокрутка

```markdown
## Проблема:
Контент подгружается при скролле

## Решение:
1. Прокрутить вниз
2. Wait для загрузки новых элементов
3. Извлечь новые данные
4. Повторить до достижения конца
5. Использовать scroll(scroll_amount="max")
```

### Защита от скрейпинга

```markdown
## Проблемы:
- CAPTCHA
- Rate limiting
- IP блокировки

## Решения:
- Уважать robots.txt
- Добавлять задержки между запросами
- Не перегружать сервер
- Использовать официальные API где доступны
```
