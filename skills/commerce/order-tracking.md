# Order Tracking (Отслеживание заказов)

## Описание

Навык мониторинга статуса доставки заказов с e-commerce платформ и транспортных компаний. Позволяет автоматически отслеживать посылки, извлекать трек-номера и предоставлять актуальную информацию о местонахождении.

## Связанные навыки

- `email-management.md` — Извлечение треков из писем
- `data-extraction.md` — Парсинг страниц отслеживания

---

## Инструменты

- **`navigate`** — Переход на страницы отслеживания
- **`get_page_text`** — Извлечение статуса доставки
- **`find`** — Поиск трек-номеров и статусов

---

## Алгоритм

### 1. Получение трек-номера
- Из email
- Из личного кабинета магазина

### 2. Определение курьерской службы
- По формату номера
- По информации в заказе

### 3. Отслеживание
- Перейти на сайт курьера
- Ввести трек-номер
- Извлечь статус и ETA

### 4. Формирование отчёта
- Текущее местоположение
- Ожидаемая дата доставки
- История перемещения

---

## Пример

```
Трек: 1Z999AA10123456784 (UPS)

Статус: In Transit
Место: Los Angeles, CA
ETA: 28 мая 2026, 18:00

История:
- 27.05 10:00 - Забран из отделения
- 27.05 14:00 - В пути
- 28.05 09:00 - В городе назначения
```

---

## Лучшие практики

1. Проверять несколько источников (магазин + курьер)
2. Сохранять историю статусов
3. Уведомлять при изменении статуса
4. Обрабатывать задержки и проблемы
5. Поддерживать множество курьеров (UPS, FedEx, DHL, USPS, и т.д.)
