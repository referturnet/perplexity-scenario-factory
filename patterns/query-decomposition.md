# Pattern: Query Decomposition

**Категория:** patterns  
**Теги:** decomposition, multi-query, parallel, research  
**Обновлено:** 2026-05-29

---

## Контекст

Сложный вопрос редко отвечается одним запросом. Декомпозиция превращает один большой вопрос в несколько точных.

## Паттерн

### Формула декомпозиции

```
[Главный вопрос] → [ЧТО?] + [КАК?] + [КТО?] + [КОГДА?] + [ПОЧЕМУ?] + [СРАВНИТЬ]
```

### Матрица подзапросов

| Ось | Шаблон запроса | Цель |
|-----|----------------|------|
| Определение | `[тема] что это` | Понять суть |
| Механизм | `[тема] как работает принцип` | Понять процесс |
| Игроки | `[тема] компании провайдеры топ` | Кто на рынке |
| Динамика | `[тема] тренды 2025 2026` | Актуальность |
| Критика | `[тема] проблемы минусы ограничения` | Риски |
| Прогноз | `[тема] будущее прогноз эксперты` | Перспективы |

## Пример

**Вопрос:** "Стоит ли инвестировать в солнечную энергетику в ЦА?"

**Декомпозиция:**
1. `солнечная энергетика Центральная Азия рынок 2025`
2. `солнечные панели Кыргызстан Казахстан проекты`
3. `инвестиции возобновляемая энергетика доходность`
4. `законодательство энергетика ЦА льготы субсидии`
5. `риски солнечная энергетика климат сезонность`
6. `прогноз энергорынок ЦА 2026 2030`

## Когда использовать

- Любой запрос сложнее "простого факта"
- Нужен отчёт с несколькими разделами
- Тема охватывает 2+ домена знаний
