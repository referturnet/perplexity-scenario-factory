# UC-008: Technology Comparison — Сравнительный анализ технологий

**Категория:** Research & Analysis (RES)  
**Сложность:** ⭐⭐⭐  
**Вывод:** TBL + RPT  
**Источник:** WEB  
**Обновлено:** 2026-05-29

## Цель

Сравнить технологии, платформы или инструменты по практическим критериям.

## Входные данные

- Список технологий или задача.
- Критерии сравнения.
- Контекст использования.

## Шаги выполнения

### Шаг 1: Определить оси сравнения
Функциональность, производительность, ecosystem, цена, ease of adoption.

### Шаг 2: Собрать данные по каждому варианту
Ключевые функции, use cases, ограничения, типичный пользователь.

### Шаг 3: Построить таблицу
Side-by-side comparison.

### Шаг 4: Дать recommendation by context
Какой вариант лучше для стартапа, enterprise, small team.

## Выходные данные

- сравнительная таблица;
- плюсы/минусы;
- рекомендации по контекстам использования.

## Пример промпта

```text
Сравни технологии: [A], [B], [C].
Оси сравнения:
- функции, стоимость, зрелость экосистемы, сложность внедрения, ограничения, лучший use case
Сделай таблицу и дай практический вывод.
```

## Критерии качества

- [ ] Есть единые оси сравнения
- [ ] Есть таблица
- [ ] Есть ограничения по каждому варианту
- [ ] Есть recommendation by context
