# Skill: Web Navigation

## Описание

Навык навигации по веб-страницам с помощью Perplexity Browser Agent.

## Возможности

- Открытие URL
- Переход по ссылкам
- Навигация назад/вперёд
- Обновление страницы
- Ожидание загрузки элементов

---

## Основные операции

### 1. Открыть URL

```markdown
Используй navigate tool:
- url: "полный URL страницы"
- tab_id: ID вкладки
```

**Пример:**
```json
{
  "url": "https://example.com",
  "tab_id": 1234567
}
```

### 2. Перейти по ссылке

```markdown
Используй computer tool с left_click:
1. Найди элемент через read_page или find
2. Нажми на него по ref или координатам
3. Дождись загрузки новой страницы
```

**Пример:**
```json
{
  "actions": [
    {"action": "left_click", "ref": "ref_123"},
    {"action": "wait", "duration": 2}
  ],
  "tab_id": 1234567
}
```

### 3. Назад/Вперёд

```markdown
Используй navigate tool:
- url: "back" или "forward"
```

**Пример:**
```json
{
  "url": "back",
  "tab_id": 1234567
}
```

### 4. Обновить страницу

```markdown
Используй computer tool с key:
- Нажать F5 или cmd+r (Mac)
```

**Пример:**
```json
{
  "actions": [
    {"action": "key", "text": "F5"}
  ],
  "tab_id": 1234567
}
```

## Продвинутые техники

### Ожидание загрузки элементов

```markdown
После навигации:
1. Используй wait (2-3 секунды)
2. Проверь наличие элементов через read_page
3. Если не загрузилось - подожди ещё
```

### Работа с SPA (одностраничные приложения)

```markdown
Для SPA:
- Клик по ссылке может не перезагружать страницу
- Используй wait + read_page для проверки изменений
- Можно проверить URL через screenshot
```

### Многостраничная навигация

```markdown
Для последовательных переходов:
1. Открыть первую страницу
2. Извлечь ссылки
3. Перейти по каждой
4. Вернуться назад (back) после каждой
```

## Лучшие практики

### ✅ Рекомендуется

1. **Всегда ждать загрузки**
   ```json
   {"action": "wait", "duration": 2}
   ```

2. **Проверять успешность навигации**
   - Использовать screenshot или read_page
   - Проверить наличие ожидаемых элементов

3. **Использовать полные URL**
   ```
   https://example.com/page
   вместо example.com/page
   ```

4. **Обрабатывать ошибки**
   - 404 страница не найдена
   - 500 ошибка сервера
   - Timeout превышение времени ожидания

### ❌ Избегать

1. **Не навигировать без ожидания**
2. **Не предполагать мгновенную загрузку**
3. **Не игнорировать ошибки навигации**
4. **Не использовать относительные URL**

## Примеры использования

### Пример 1: Простой переход

```markdown
1. Открой https://example.com
2. Подожди 2 секунды
3. Проверь загрузку
```

### Пример 2: Навигация с кликом

```markdown
1. Найди ссылку "Контакты"
2. Кликни по ней
3. Дождись загрузки новой страницы
4. Проверь, что URL изменился
```

### Пример 3: Многостраничный обход

```markdown
1. Открой список статей
2. Извлеки все URL статей
3. Для каждого URL:
   - Перейди по ссылке
   - Извлеки данные
   - Вернись назад
4. Сохрани все данные
```

## Edge Cases

### 1. Медленные сайты
```markdown
Увеличь wait до 5-10 секунд
Используй повторные проверки
```

### 2. Попапы и модальные окна
```markdown
Проверяй наличие попапов после навигации
Закрывай их перед продолжением
```

### 3. Редиректы
```markdown
URL может измениться после навигации
Проверяй финальный URL через screenshot
```

### 4. Аутентификация
```markdown
Проверяй, не требуется ли вход
Используй /patterns/authentication/login-flow.md
```
