# Product Search (Поиск продуктов)

## Описание

Навык автоматического поиска товаров на e-commerce платформах с применением фильтров, сортировки и анализа результатов. Позволяет AI-агенту находить оптимальные варианты товаров по заданным критериям, извлекать характеристики, цены, отзывы и формировать сравнительные таблицы для принятия решений.

## Связанные навыки

- `price-comparison.md` — Сравнение цен найденных товаров
- `data-extraction.md` — Извлечение структурированных данных о товарах
- `navigation.md` — Навигация по каталогам и категориям
- `form-filling.md` — Заполнение поисковых форм с фильтрами

---

## Инструменты

### Доступные функции:

- **`search_web(queries)`** — Поиск товаров через поисковые системы
- **`navigate(url, tab_id)`** — Переход на страницы каталогов и карточек товаров
- **`get_page_text(query, tab_id)`** — Извлечение информации о товарах
- **`read_page(tab_id)`** — Анализ структуры страницы каталога
- **`find(query, tab_id)`** — Поиск конкретных элементов (цены, кнопки фильтров)
- **`form_input(ref, value, tab_id)`** — Установка фильтров и параметров поиска
- **`computer(actions, tab_id)`** — Клики по фильтрам, сортировке, пагинации
- **`tabs_create(url)`** — Параллельный поиск на нескольких площадках

## Алгоритм выполнения

### 1. Анализ запроса и подготовка
- Разобрать требования: категория, бренд, характеристики, ценовой диапазон
- Определить приоритетные площадки (Amazon, Ozon, Wildberries, AliExpress, местные)
- Сформулировать поисковые запросы с учётом синонимов и вариантов написания
- Открыть несколько вкладок для параллельного поиска на разных платформах

### 2. Навигация к разделу поиска
- Перейти на главную страницу площадки
- Найти поисковую строку через `find` или `read_page`
- Ввести базовый поисковый запрос
- Дождаться загрузки результатов

### 3. Применение фильтров
- Идентифицировать панель фильтров на странице
- Последовательно применить фильтры:
  - Ценовой диапазон (min/max)
  - Бренд/производитель
  - Технические характеристики (размер, цвет, материал)
  - Рейтинг и наличие отзывов
  - Условия доставки
- После каждого изменения дождаться обновления результатов

### 4. Сортировка результатов
- Применить сортировку по релевантности:
  - По популярности
  - По цене (возрастание/убывание)
  - По рейтингу
  - По количеству отзывов
- Зафиксировать выбранный критерий сортировки

### 5. Сбор данных о товарах
- Прокрутить страницу для загрузки всех товаров (учесть lazy loading)
- Извлечь для каждого товара:
  - Название и артикул
  - Цену (текущую, старую, скидку)
  - Рейтинг и количество отзывов
  - Основные характеристики из превью
  - Ссылку на карточку товара
  - Доступность и срок доставки
- Перейти на следующие страницы пагинации при необходимости

### 6. Детальный анализ топовых вариантов
- Открыть карточки 5-10 наиболее подходящих товаров
- Извлечь полные спецификации
- Проанализировать отзывы (особенно негативные)
- Проверить условия возврата и гарантии
- Зафиксировать все варианты цены (с купонами, по подписке)

### 7. Формирование результата
- Создать сравнительную таблицу найденных товаров
- Ранжировать по соответствию критериям
- Указать преимущества и недостатки каждого варианта
- Добавить прямые ссылки на товары

## Примеры использования

### Пример 1: Поиск ноутбука по критериям
```
Запрос: "Найти ноутбук 15.6\" с 16GB RAM, SSD 512GB, бюджет $800-1200"

Действия:
1. Открыть Amazon, Best Buy, Newegg в отдельных вкладках
2. Ввести "laptop 15.6 inch 16GB 512GB SSD"
3. Применить фильтры:
   - Price: $800-$1200
   - Screen Size: 15.6"
   - RAM: 16GB
   - Storage: 512GB SSD
4. Сортировка: по рейтингу
5. Собрать топ-10 моделей с каждой площадки
6. Сравнить CPU, GPU, вес, время работы

Результат: Таблица с 15 лучшими вариантами и рекомендациями
```

### Пример 2: Поиск смартфона по бренду
```
Запрос: "Найти Samsung Galaxy S-серии последних 2 лет"

Действия:
1. Открыть samsung.com, Amazon, местные ритейлеры
2. Фильтр: Brand = Samsung, Series = Galaxy S
3. Сортировка по дате выпуска (новые впереди)
4. Собрать S23, S23+, S23 Ultra, S24, S24+, S24 Ultra
5. Проверить текущие цены и акции

Результат: Список с ценами, характеристиками, ссылками
```

### Пример 3: Поиск товара с лучшими отзывами
```
Запрос: "Найти беспроводные наушники с рейтингом 4.5+ и 100+ отзывов"

Действия:
1. Поиск: "wireless earbuds"
2. Фильтры:
   - Rating: 4.5 stars & up
   - Min Reviews: 100
3. Сортировка по количеству отзывов
4. Открыть топ-5, прочитать негативные отзывы
5. Сравнить частые жалобы

Результат: Топ-5 с анализом плюсов/минусов по отзывам
```

## Edge cases и обработка ошибок

### Проблема: Динамическая загрузка (lazy loading)
```markdown
Решение:
- Прокручивать страницу постепенно, дожидаясь загрузки новых товаров
- Использовать scroll с паузами по 2-3 секунды
- Проверять изменение высоты страницы или числа элементов
- Остановиться при достижении конца или максимального количества
```

### Проблема: Фильтры не применяются мгновенно
```markdown
Решение:
- Дожидаться AJAX-запроса после каждого клика
- Проверять изменение URL или количества товаров
- Использовать wait на 2-5 секунд перед следующим действием
- Проверять наличие индикаторов загрузки
```

### Проблема: Цены скрыты или требуют входа
```markdown
Решение:
- Сообщить пользователю о необходимости авторизации
- Попробовать региональные версии сайта
- Использовать агрегаторы цен (напр. CamelCamelCamel)
- Отметить товары с недоступной ценой
```

### Проблема: Разная структура на площадках
```markdown
Решение:
- Адаптировать алгоритм под каждую платформу (Amazon, eBay, Ozon имеют разные UI)
- Использовать `find` для поиска элементов по назначению
- Создать фолбэк на ручную навигацию при неудаче
```

### Проблема: Товар закончился (оут-оф-сток)
```markdown
Решение:
- Отметить статус доступности в результатах
- Искать альтернативные предложения на этой же платформе
- Проверить другие площадки
- Сохранить ссылку для отслеживания появления
```

## Метрики качества

- **Полнота результатов**: найдено минимум 10 подходящих вариантов
- **Точность фильтрации**: все товары соответствуют критериям
- **Полнота данных**: извлечены все ключевые характеристики (>90%)
- **Актуальность цен**: цены проверены в течение последних 24 часов
- **Покрытие площадок**: проверены минимум 3 крупные платформы
- **Скорость поиска**: завершение за <3 минуты на площадку

---

## Лучшие практики

1. **Параллельный поиск** — открывать несколько площадок одновременно
2. **Применять фильтры последовательно** — от общего к частному
3. **Сохранять промежуточные результаты** — чтобы не потерять данные при сбоях
4. **Проверять доступность** — не все товары в каталоге есть в наличии
5. **Считывать все варианты цен** — с купонами, подписками, промокодами
6. **Обращать внимание на негативные отзывы** — они часто важнее позитивных
7. **Учитывать сроки доставки** — для некоторых товаров это критично
8. **Сравнивать спецификации детально** — не полагаться только на названия
9. **Проверять акции и скидки** — искать промокоды, сезонные распродажи
10. **Создавать структурированные таблицы** — для удобства сравнения
