# UC-002: Competitive Analysis — Конкурентный анализ продукта

**Категория:** Research & Analysis (RES)  
**Сложность:** ⭐⭐⭐  
**Вывод:** TBL (Comparison Table) + RPT  
**Источник:** WEB  
**Обновлено:** 2026-05-29

---

## 🎯 Цель

Собрать и структурировать данные о конкурентах: функции, цены, позиционирование, слабые стороны.

---

## 📥 Входные данные

- Название продукта/ниши
- Список конкурентов (или "найди сам")
- Оси сравнения (цена / функции / аудитория / UX)
- Регион (глобальный / СНГ / ...)

---

## 🔄 Шаги выполнения

### Шаг 1: Идентификация конкурентов

```
🔍 "[ниша] лучшие инструменты 2025 2026"
🔍 "[продукт] альтернативы конкуренты"
🔍 "[ниша] топ компании рейтинг"
```

### Шаг 2: Сбор данных по каждому конкуренту

Для каждого из топ-5:
```
🔍 "[конкурент] цены тарифы 2025"
🔍 "[конкурент] функции возможности"
🔍 "[конкурент] отзывы проблемы"
```

### Шаг 3: Составление таблицы

```markdown
| Параметр | Продукт A | Продукт B | Продукт C |
|----------|-----------|-----------|----------|
| Цена     |           |           |          |
| Функции  |           |           |          |
| Аудитория|           |           |          |
| Сильные  |           |           |          |
| Слабые   |           |           |          |
```

### Шаг 4: Анализ позиционирования

- Где незанятая ниша?
- Кто лидер по цене / функциям / UX?
- Какие паттерны повторяются?

---

## 📤 Выходные данные

- Таблица сравнения (min 5 конкурентов × 6 осей)
- SWOT-краткая для каждого
- Выводы: где возможность

- [ ] Раздел "выводы и возможности"
